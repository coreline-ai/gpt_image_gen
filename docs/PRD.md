# 📌 Product Requirements Document (PRD)
## Project Name
IMAGE_PROJECT_MANAGE

## 1. 목표 (Goal)
사용자가 자연어로 프로젝트를 조회하고,
MCP에 등록된 이미지 생성 프로젝트 내용을 확인한 뒤,
Custom GPT가 직접 이미지 생성 작업을 실행하고,
결과를 WebApp Server로 자동 업로드하여 관리하는 시스템 구축

## 2. 핵심 가치
- GPT 기반 작업 오케스트레이션
- MCP 기반 프로젝트/프롬프트 조회
- Custom GPT 기반 이미지 직접 생성
- 생성 직후 자동 업로드
- WebApp 기반 결과 확인
- Web App 기반 상태/결과 시각화

## 3. 주요 사용자 시나리오

### 3.1 프로젝트 조회
사용자:
"현재 프로젝트 조회"

시스템:
- MCP: list_active_projects 호출
- GPT: 프로젝트 목록과 요약 표시

---

### 3.2 프로젝트 실행 (이미지 생성)

사용자:
"수행해"

시스템:
1. MCP: list_active_projects 호출 또는 현재 선택된 프로젝트 사용
2. MCP: `get_project_execution_context`로 프로젝트 상세 내용과 이미지 생성 프롬프트 조회
3. GPT: 프로젝트 내용, 생성 대상, 프롬프트를 사용자에게 표시하고 확인 요청
4. Custom GPT: 내장 Image Generation으로 이미지를 직접 생성
5. Custom GPT Action/MCP: 생성 완료 직후 프로젝트에 연결된 업로드 URL로 자동 업로드
6. 서버: WebApp Server가 정한 디렉터리에 이미지 저장
7. GPT: 저장 결과와 파일 경로/URL 요약

---

### 3.3 일괄 이미지 생성

사용자:
"24개 모두 생성해"

시스템:
- MCP: list_active_projects 호출 또는 현재 선택된 프로젝트 사용
- GPT: 프로젝트 요약 및 전체 실행 확인
- MCP: 각 아이템의 생성 프롬프트와 업로드 정보를 순차 조회
- Custom GPT: 첫 번째 아이템을 먼저 생성하고 즉시 자동 업로드
- GPT: 첫 번째 결과 업로드 후 나머지 아이템 계속 처리 여부를 확인
- 사용자가 계속을 선택하면 남은 아이템을 같은 방식으로 순차 처리 및 자동 업로드
- GPT: 아이템별 저장 결과 및 남은 작업 상태 출력

## 6. 성공 기준

- MCP 프로젝트 조회 성공
- GPT가 프로젝트 내용/프롬프트 표시 성공
- GPT 내부 이미지 생성 성공
- 이미지 생성 → 자동 업로드 → 경로 저장 성공
- Web App에서 상태 확인 가능
- Web App에서 버튼 클릭으로 업로드된 이미지 확인 가능
- 로그 기록 정상 작동

---

## 7. 비기능 요구사항

- 응답 시간 < 3초 (단일 요청)
- 배치 처리 안정성
- 오류 시 재시도 가능

---

## 8. 운영 원칙

- 이미지 생성은 반드시 Custom GPT에서만 수행한다
- MCP/서버는 이미지 생성 자체를 수행하지 않는다
- Custom GPT는 생성 전에 MCP에서 프로젝트 내용과 이미지 생성 프롬프트를 읽어온다
- GPT는 생성 전에 프로젝트 내용을 사용자에게 보여주고 확인을 받는다
- 생성이 완료되면 별도 수동 저장 단계 없이 즉시 자동 업로드한다
- 저장 완료 후 파일 경로 또는 공개 가능한 HTTP(S) URL을 사용자와 Web App에 제공한다
- 배치 요청은 전체를 즉시 일괄 처리하지 않고, 첫 번째 아이템을 먼저 처리한 뒤 나머지 계속 여부를 확인하는 순차형 흐름을 기본으로 한다

---

## 9. 데모 완료 기준

- Custom GPT에서 프로젝트 목록을 읽어올 수 있다
- 선택한 프로젝트의 이미지 생성 프롬프트를 Custom GPT에서 보여줄 수 있다
- 사용자가 확인하면 Custom GPT가 이미지를 직접 생성한다
- 생성 결과가 WebApp Server 디렉터리로 자동 업로드된다
- Web App에서 업로드된 이미지를 버튼으로 확인할 수 있다
